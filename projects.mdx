# Project

A project is where you bring together devices, datasets, training, and deployment for a single model.

---

## Overview

Everything in EdgeStudio happens inside a project. When you create one, you pick a task type — **Image Classification**, **Object Detection**, or **Audio Anomaly Detection** — which determines the training options and metrics available later on.

---

## Creating a Project

From the [Dashboard](/docs/dashboard), click **New Project**, give it a name, and choose a task type. The task type can't be changed after creation, so pick the one that matches what you're trying to build.

---

## Managing a Project

From the Projects table you can:

- **Open** a project to work on its [Devices](/docs/devices), [Datasets](/docs/datasets), [Training](/docs/training), and [Deployment](/docs/deployment)
- **Edit** a project's name/description
- **Delete** a project

---

## Personal vs. Organization Projects

Projects live in whichever workspace you created them in. Switch workspaces from the [Dashboard](/docs/dashboard) to see personal projects vs. an organization's projects — they're kept separate.

---

## Troubleshooting

- **I can't find a project I created.**
  Check you're in the right workspace (Personal vs. Organization) using the switcher on the Dashboard.

- **I picked the wrong task type.**
  Task type can't be changed after creation — create a new project with the correct type instead.
